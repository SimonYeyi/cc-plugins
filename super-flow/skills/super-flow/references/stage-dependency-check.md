# 阶段依赖检查（防止跳步）

**强制规则**：在进入任何阶段前，必须验证上一阶段已完成并通过评审。

## 阶段依赖关系

```
Brainstorming → SPEC生成
SPEC生成完成 → SPEC确认
SPEC确认通过 → SPEC评审
SPEC评审通过 → 设计流程
设计评审通过 → 架构流程
架构评审通过 → 开发流程
开发评审通过 → 测试流程
```

## 验证方法

1. 检查上一阶段的输出文档是否存在
2. 检查评审报告是否显示"通过"
3. 如果任一检查失败，**主动回退并补全流程**

## 自动纠正流程

如果发现跳步，主控必须：

1. **识别缺失的阶段**：确定哪个前置阶段未完成
2. **主动回退**：暂停当前阶段，回到缺失阶段
3. **补全流程**：执行缺失阶段的完整流程（包括评审）
4. **继续原计划**：补全后回到原计划的阶段

### 示例

```
用户请求：启动设计Agent
主控检查：发现SPEC评审未通过
主控行动：
  1. 识别：缺少SPEC评审通过状态
  2. 回退：暂停设计流程
  3. 补全：启动SPEC评审Agent → 如果不通过则循环修复
  4. 继续：SPEC评审通过后，再启动设计Agent
```
